üì¶ Quick Start Integration

Step 1: Include the CSS and Script

Add the weather widget CSS in the <head> section:

<link rel="stylesheet" href="path/to/weather-widget.css">

Add the weather widget script before the closing </body> tag:

<script src="path/to/weather-widget.js"></script>

Step 2: Add the Custom Element

Place the <weather-widget> custom element anywhere in your HTML:

<weather-widget></weather-widget>

Step 3: That's It! üéâ

The widget will automatically:

  • Request geolocation permission on first load (if no saved cities)
  • Display weather for the user's location or saved cities
  • Save user preferences to browser LocalStorage
  • Refresh weather data every 10 minutes
Note: The widget uses Shadow DOM for style encapsulation, so it won't conflict with your page's CSS and your styles won't affect the widget.

üé® Customization

The widget uses Shadow DOM for style encapsulation. To customize the container around the widget, simply style the parent element:

<style>
    .my-widget-container {
        max-width: 400px;
        margin: 20px auto;
        padding: 20px;
        background: white;
        border-radius: 8px;
        box-shadow: 0 2px 8px rgba(0,0,0,0.1);
    }
</style>

<div class="my-widget-container">
    <weather-widget></weather-widget>
</div>

üîß Troubleshooting

  • Widget not appearing? Make sure the script is loaded and the custom element tag is correct.
  • Geolocation not working? The user must grant permission, and the page must be served over HTTPS (except localhost).
  • Weather not loading? Check browser console for API errors. The widget requires an OpenWeather API key.
  • Cities not saving? Ensure LocalStorage is enabled in the browser (not in private/incognito mode).
